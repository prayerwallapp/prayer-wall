## Surfaces flagged NO_DESIGN — need Josiah's direction

These exist in code but have no Figma counterpart. Do not style them without explicit direction:

1. **ProfileModal** (edit profile sheet) — substantial real functionality (photo upload, name edit, password change, notification prefs, GDPR data export + account deletion). Likely needs a Figma design before a token pass makes sense; styling it arbitrarily would introduce patterns that conflict with whatever gets designed later.
2. **NotificationBell + dropdown** — real-time subscription, mark-read logic. Could be styled to match the Profile Menu dropdown treatment but there's no Figma spec.
3. **Landing page** — marketing surface with its own aesthetic. Different design context than the app.
4. **Admin layout sidebar** — navigation chrome for the admin area. If the Moderation Inbox shell in Figma includes a sidebar, it would resolve this.
5. **Admin moderation actions** (approve/hold/reject buttons) — status color semantics (emerald/amber/red) have no token equivalents yet; needs a product decision before adding semantic status tokens.
6. **Admin keyword rules page** — no Figma panel.
7. **Admin digest settings page** — no Figma panel.
8. **Auth reset form** — infrequently used, low priority.
9. **Embed wall + SubmissionsGrid** — embed is a special surface. Worth deciding whether it should mirror the main wall or have its own lighter treatment.
10. **Toast/notification system** — doesn't exist; needs a design and build decision.
11. **Error boundaries / 404 pages** — not implemented; Next.js default fallbacks active.
12. **Upgrade prompts** — no reusable component; display page has an ad-hoc one.
